# task type: communication-summary

## 対象

要約、報告文、ユーザー向け説明、引き継ぎ文、質問整理、説明メモ。

## 使う場面

- ユーザー向け説明や引き継ぎ文、報告文を作るとき
- 調査結果や実施内容を、読み手向けに要約して伝えるとき
- 次アクションや確認質問を含む説明メモを整えるとき

## 基本方針

- 読み手を明確にする
- 事実と判断を分ける
- 次アクションを明示する
- 不足情報があるなら質問として切り出す

## preflight

- 読み手、目的、要約対象の範囲は明確か
- どの source まで追えていて、coverage limit がどこにあるか分かるか
- 事実 / 判断 / 未確認を分けて書ける材料があるか
- 次アクションの owner が分かるか。不明なら質問や保留で返せるか
- 正本文書更新、追加調査、読み手向け要約のどれが主目的か区別できるか

## 推奨する構成

- 背景 / 目的
- 事実
- 判断または提案
- 次アクション
- 必要なら質問

## 報告で重視すること

- 何向けの文章を作ったか
- 何を根拠にまとめたか
- どこが事実で、どこが判断か
- 相手が次に何をすればよいか
- coverage limit や未回収論点がどこにあるか
- owner 不明の action を勝手に確定していないか
- 不足情報を質問として切り出せているか

## continue / stop / partial-complete の例

- continue: 読み手、目的、source 範囲が明確で、事実と判断を分けて要約できる
- continue: coverage limit はあるが、未回収点を質問として切り出したうえで、確認済み範囲の要約を返せる
- stop: 要約対象 source が不足または衝突しており、安全な説明文を作れない
- stop: 正本文書の更新や追加調査がないと成立せず、communication-summary の範囲を超える
- partial-complete: 一部 thread / 資料だけを要約し、coverage limit と未回収論点を明示して返す
- partial-complete: 次アクションは整理できたが、owner 不明項目は質問または親判断待ちとして返す

## 境界例

- 読み手向けに背景 / 事実 / 判断 / 次アクションを整理する: communication-summary
- 正本の current / proposal を確定文として整理・更新する: documentation が主
- 追加 source を探索し、事実関係を深掘りする: research が主
- communication-summary では、未確定事項を仕様確定のように昇格させない

## 注意点

- 断定しすぎない
- 専門用語や略語は読み手に合わせる
- 元資料と矛盾する説明を作らない
- coverage limit を隠さない
- owner 不明の action item は勝手に割り当てず、質問または保留で返す
